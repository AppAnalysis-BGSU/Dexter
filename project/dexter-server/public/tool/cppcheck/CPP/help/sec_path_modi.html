Checker Code
  sec_path_modi
Description
  hacker would put a symbolic link
Severity
Critical
Detector / Bug Pattern
fopen(file_name, "w");
More Information
you should check symbolic link first (using lstat or realpath)
hacker can put a symlink and exploit it to modify system files(/etc/passwd, /etc/init.d/xxx, /etc/samba/smb.conf).
Bad Code 
fp = fopen(path, "w");
Good Code 
------------------- positive example 1 -------------------
if (realpath(path,pResPath) == NULL) {
  printf("realpath error\n");
  return;
}

if (!strcmp(path, pResPath)) {
  fp = fopen(path, "w");
}


------------------- positive example 2 -------------------
struct stat lstat_info;
if(lstat(filename, &lstat_info) == -1)
{
  printf("lstat err");
  return;
}

if (!S_ISLNK(lstat_info.st_mode))
{
  fp = fopen(path, "w");
}
